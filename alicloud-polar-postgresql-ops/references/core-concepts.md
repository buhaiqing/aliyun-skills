# Core Concepts

> PolarDB PostgreSQL 核心概念和架构

## Architecture Overview

PolarDB PostgreSQL 采用**计算-存储分离**架构：

```
┌─────────────────────────────────────────────────────────────────┐
│                     PolarDB PostgreSQL 架构                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────┐    ┌─────────────────┐                   │
│   │   Primary Node  │◄───│  Read-Only Node │  计算层 (Compute)  │
│   │   (读写节点)     │    │   (只读节点)     │                   │
│   │   - PostgreSQL  │    │   - PostgreSQL  │                   │
│   │   - Buffer Pool │    │   - Buffer Pool │                   │
│   │   - WAL         │    │   - WAL Replay  │                   │
│   └────────┬────────┘    └────────┬────────┘                   │
│            │                      │                             │
│            │   RDMA / PolarFS     │                             │
│            ▼                      ▼                             │
│   ┌─────────────────────────────────────────┐                  │
│   │           PolarFS 存储层                 │  存储层 (Storage) │
│   │   - Shared Storage (共享存储)           │                  │
│   │   - Redo Log                            │                  │
│   │   - Data Pages                          │                  │
│   │   - 多副本同步复制                       │                  │
│   └─────────────────────────────────────────┘                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Key Concepts

### 1. 计算-存储分离 (Compute-Storage Separation)

| 特性 | 说明 | 优势 |
|------|------|------|
| 计算层 | 运行 PostgreSQL 引擎的节点 | 独立扩缩容 |
| 存储层 | PolarFS 分布式文件系统 | 数据共享，秒级扩容 |
| RDMA | 远程直接内存访问 | 低延迟存储访问 |

### 2. 读写分离 (Read/Write Splitting)

```
┌─────────────────────────────────────────┐
│           读写分离架构                   │
├─────────────────────────────────────────┤
│                                         │
│   ┌─────────────┐    ┌─────────────┐   │
│   │ Application │    │ Application │   │
│   └──────┬──────┘    └──────┬──────┘   │
│          │                  │           │
│          ▼                  ▼           │
│   ┌─────────────────────────────────┐   │
│   │      Cluster Endpoint           │   │
│   │      (集群访问点)                │   │
│   │   - 自动读写分离                 │   │
│   │   - 负载均衡                     │   │
│   └──────────────┬──────────────────┘   │
│                  │                      │
│          ┌───────┴───────┐              │
│          ▼               ▼              │
│   ┌─────────────┐  ┌─────────────┐     │
│   │   Primary   │  │    Read     │     │
│   │   (写请求)   │  │   Replica   │     │
│   └─────────────┘  │  (读请求)    │     │
│                    └─────────────┘     │
│                                         │
└─────────────────────────────────────────┘
```

### 3. Serverless (无服务器模式)

| 参数 | 说明 | 范围 |
|------|------|------|
| `ScaleMin` | 最小计算单元 | 1-32 RCU |
| `ScaleMax` | 最大计算单元 | 1-1000 RCU |
| `ScaleRoNumMin` | 最小只读节点数 | 0-15 |
| `ScaleRoNumMax` | 最大只读节点数 | 0-15 |

**RCU (PolarDB Capacity Unit)**：
- 1 RCU ≈ 1 核 + 4GB 内存
- 按秒计费，精确到 0.5 RCU
- 自动扩缩容，秒级响应

### 4. GDN (Global Database Network)

```
┌─────────────────────────────────────────────────────────────────┐
│                      GDN 全球数据库网络                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────┐        ┌─────────────────┐               │
│   │  Region: 杭州    │◄──────►│  Region: 北京    │               │
│   │  - Primary      │  Sync  │  - Replica      │               │
│   │  - Read/Write   │        │  - Read Only    │               │
│   └─────────────────┘        └─────────────────┘               │
│           │                           │                         │
│           │ Async                     │ Async                   │
│           ▼                           ▼                         │
│   ┌─────────────────┐        ┌─────────────────┐               │
│   │  Region: 香港    │◄──────►│  Region: 新加坡  │               │
│   │  - Replica      │  Sync  │  - Replica      │               │
│   │  - Read Only    │        │  - Read Only    │               │
│   └─────────────────┘        └─────────────────┘               │
│                                                                 │
│   同步复制 (Sync): RPO = 0                                      │
│   异步复制 (Async): RPO > 0, 跨地域                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5. Endpoint 类型

| Endpoint 类型 | 用途 | 特性 |
|--------------|------|------|
| **Primary Endpoint** | 连接主节点 | 读写，高可用自动切换 |
| **Cluster Endpoint** | 集群访问点 | 自动读写分离，负载均衡 |
| **Custom Endpoint** | 自定义访问点 | 指定节点，自定义读写模式 |

### 6. 节点类型

| 节点类型 | 角色 | 用途 |
|----------|------|------|
| **Primary Node** | 主节点 | 处理读写请求，数据变更 |
| **Read-Only Node** | 只读节点 | 处理读请求，扩展读能力 |

```

│                       PolarFS 存储架构                           │

│                                                                 │

│   │                    Chunk Server 层                       │  │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │  │
│   │  │ Chunk 1 │  │ Chunk 2 │  │ Chunk 3 │  │ Chunk N │    │  │
│   │  │ (副本A) │  │ (副本A) │  │ (副本A) │  │ (副本A) │    │  │
│   │  └─────────┘  └─────────┘  └─────────┘  └─────────┘    │  │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │  │
│   │  │ Chunk 1 │  │ Chunk 2 │  │ Chunk 3 │  │ Chunk N │    │  │
│   │  │ (副本B) │  │ (副本B) │  │ (副本B) │  │ (副本B) │    │  │
│   │  └─────────┘  └─────────┘  └─────────┘  └─────────┘    │  │
│   │  ┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐    │  │
│   │  │ Chunk 1 │  │ Chunk 2 │  │ Chunk 3 │  │ Chunk N │    │  │
│   │  │ (副本C) │  │ (副本C) │  │ (副本C) │  │ (副本C) │    │  │
│   │  └─────────┘  └─────────┘  └─────────┘  └─────────┘    │  │
│   └─────────────────────────────────────────────────────────┘  │
│                              ▲                                  │
│                              │ RDMA                              │
│   ┌──────────────────────────┴───────────────────────────┐     │
│   │                    PolarSwitch 层                     │     │
│   │              (存储网关，路由请求到 Chunk)              │     │
│   └─────────────────────────────────────────────────────────┘  │
│                              ▲                                  │
│                              │                                  │
│   ┌──────────────────────────┴───────────────────────────┐     │

│   - 三副本强同步 (2+1)                                          │
│   - 自动故障恢复                                                │
│   - 在线扩容 (秒级)                                             │
│   - 按量付费                                                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## Parameter Groups

### 关键参数分类

| 类别 | 参数示例 | 说明 |
|------|----------|------|
| **连接** | max_connections | 最大连接数 |
| **内存** | shared_buffers | 共享缓冲区大小 |
| **WAL** | wal_level | WAL 日志级别 |
| **复制** | max_replication_slots | 最大复制槽数 |
| **并行** | max_parallel_workers | 最大并行工作线程 |

## Backup Types

| 备份类型 | 描述 | 恢复时间 |
|----------|------|----------|
| **全量备份** | 完整数据备份 | 分钟级 |
| **增量备份** | 基于上次备份的增量 | 分钟级 |
| **日志备份** | WAL 日志连续备份 | 秒级 (PITR) |

## Performance Metrics

### 关键性能指标

| 指标 | 说明 | 健康阈值 |
|------|------|----------|
| CPUUtilization | CPU 使用率 | < 70% |
| MemoryUtilization | 内存使用率 | < 80% |
| ConnectionUsage | 连接使用率 | < 80% |
| IOPSUsage | IOPS 使用率 | < 80% |
| DiskUsage | 磁盘使用率 | < 85% |
| BufferHitRate | 缓冲区命中率 | > 95% |
