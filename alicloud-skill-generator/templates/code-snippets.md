# Code Snippets 模板（仅 `cli_applicability: sdk-only` 适用）

> **Purpose:** 为 CLI 不足以覆盖完整功能、必须依赖 SDK（API 方式）的 skill，生成
> 可直接 `go run` 的标准化 Go SDK 代码片段。每个 Operation 一个独立 `.go` 文件，附带
> 独立 `go.mod` + `README.md`，便于 Agent 或用户在沙箱里独立编译运行。
>
> **适用条件：** `cli_applicability: sdk-only` （CLI 完全不暴露该 product/operation）
>
> **生成时机：** [Evaluation-Driven Generation Workflow §Step 5.5](../SKILL.md#step-55-generate-code-snippets-when-sdk-only)

---

## 1. 目录结构

```
alicloud-[product]-ops/
└── assets/
    └── code-snippets/                # 仅 sdk-only 适用，dual-path/cli-first 不生成
        ├── go.mod                    # 独立 module, 路径示例: aliyun-skills/alicloud-[product]-ops/assets/code-snippets
        ├── go.sum                    # go mod tidy 生成
        ├── README.md                 # 使用说明 + 凭据环境变量 + 运行步骤
        ├── main.go                   # 公共 client 初始化 (可被各 snippet 引用)
        ├── create_<resource>.go      # 每个 operation 一个 .go 文件
        ├── describe_<resource>.go
        ├── modify_<resource>.go
        ├── delete_<resource>.go
        └── ...                       # 其它 product-specific operation
```

**文件命名规范：**
- `<verb>_<resource>.go`（如 `create_instance.go`、`describe_db_instances.go`）
- 全部小写、下划线分隔
- 动词需与 OpenAPI OperationId 的动词部分对齐（Create / Describe / Modify / Delete / List / Get / Start / Stop / Restart / Upgrade 等）
- 资源名用单数（如 `db_instance` 而非 `db_instances`），除非 OpenAPI 明确为复数

## 2. 通用脚手架：`main.go`

每个 code snippet 引用同一个 `main.go` 提供的 `Client` 工厂，避免重复初始化逻辑。

```go
// Code generated by alicloud-skill-generator; DO NOT EDIT manually.
// Reference: references/integration.md §Go Runtime Bootstrap
package snippets

import (
    "fmt"
    "os"

    openapi "github.com/alibabacloud-go/darabonba-openapi/v2/client"
    "<product-package>/<version>/client"   // 例如 rds-20140815/v2/client
)

// NewClient constructs a [Product] client using env credentials.
// MUST be the only place credentials are referenced; never log SK.
func NewClient(endpoint string) (*client.Client, error) {
    ak := os.Getenv("ALIBABA_CLOUD_ACCESS_KEY_ID")
    sk := os.Getenv("ALIBABA_CLOUD_ACCESS_KEY_SECRET")
    if ak == "" || sk == "" {
        return nil, fmt.Errorf("missing ALIBABA_CLOUD_ACCESS_KEY_ID or ALIBABA_CLOUD_ACCESS_KEY_SECRET")
    }

    config := &openapi.Config{
        AccessKeyId:     &ak,
        AccessKeySecret: &sk,
        Endpoint:        &endpoint,
    }
    return client.NewClient(config)
}
```

> **强制约束：**
> - **绝不**在代码中硬编码、打印或回显 `ALIBABA_CLOUD_ACCESS_KEY_SECRET`（遵守 credential-masking）
> - Endpoint 优先用 `os.Getenv("ALIBABA_CLOUD_REGION_ID")` 拼接区域 endpoint
> - Client 工厂签名统一为 `NewClient(endpoint string) (*client.Client, error)`

## 3. Operation Snippet 模板：`<verb>_<resource>.go`

```go
// Code generated by alicloud-skill-generator; DO NOT EDIT manually.
// Operation:    <OperationId>           (e.g., CreateDBInstance)
// SDK Method:   <Client>.<MethodName>   (e.g., client.CreateDBInstance)
// API Version:  <YYYY-MM-DD>
// CLI Support:  sdk-only (no aliyun CLI equivalent)
// OpenAPI Doc:  <official doc URL>
//
// Usage:
//   cd alicloud-[product]-ops/assets/code-snippets
//   ALIBABA_CLOUD_REGION_ID=cn-hangzhou go run create_<resource>.go

package main

import (
    "encoding/json"
    "fmt"
    "log"
    "os"

    "<product-package>/<version>/client"
    "github.com/alibabacloud-go/tea/tea"
)

func main() {
    // 1. 构造请求 (字段名与 OpenAPI Request Schema 1:1 对齐)
    req := &client.<RequestType>{
        RegionId:            tea.String(os.Getenv("ALIBABA_CLOUD_REGION_ID")),
        // ... 其它必填/选填参数
    }

    // 2. 调用 Client
    c, err := snippets.NewClient("<product>." + os.Getenv("ALIBABA_CLOUD_REGION_ID") + ".aliyuncs.com")
    if err != nil {
        log.Fatalf("client init: %v", err)
    }

    resp, err := c.<MethodName>(req)
    if err != nil {
        // 错误信息必须包含 RequestId, 便于回溯
        log.Fatalf("<OperationId> failed: %v", err)
    }

    // 3. 序列化输出 (便于 Agent 解析)
    out, _ := json.MarshalIndent(resp, "", "  ")
    fmt.Println(string(out))

    // 4. 关键字段提示 (供 Agent 提取)
    // $.DBInstanceId   -> 新建实例 ID
    // $.RequestId      -> 用于工单/日志关联
}
```

**模板字段填写检查表：**

| 字段 | 来源 | 必填 |
|------|------|------|
| `<OperationId>` | OpenAPI OperationId | ✓ |
| `<RequestType>` | SDK Go struct（`&client.<RequestType>{}`） | ✓ |
| `<MethodName>` | SDK method on `*client.Client` | ✓ |
| `<product-package>/<version>` | `go.mod` 依赖，与 `references/integration.md` 对齐 | ✓ |
| 必填参数 | OpenAPI Request Schema 中 `required: true` 字段 | ✓ |
| Endpoint | `<product>.<region>.aliyuncs.com` 或 `<product>.aliyuncs.com` (global) | ✓ |

### 🚨 P0 硬约束：所有参数必须从环境变量读取

**绝不能**在 snippet 中硬编码任何业务参数（region、instance name、version、规格、数量等）。
唯一允许写死的只有：

| 允许写死 | 原因 |
|---------|------|
| `endpoint`（单 region 产品如 DAS） | 产品约束，但 `RegionId` 字段仍从 env 读 |
| 操作类型常量（如 `Method=POST`） | HTTP 协议层 |

**所有**动态参数（含 region、instance ID、规格、数量、密码等）一律从 `os.Getenv()` 读取：

```go
// ✅ GOOD — 全部从 env 读
req := &das.CreateCacheAnalysisJobRequest{
    RegionId:   tea.String(os.Getenv("ALIBABA_CLOUD_REGION_ID")),
    InstanceId: tea.String(os.Getenv("INSTANCE_ID")),
    NodeId:     tea.String(os.Getenv("NODE_ID")),
}

// ❌ BAD — 硬编码 region
req := &das.CreateCacheAnalysisJobRequest{
    RegionId:   tea.String("cn-shanghai"),  // 禁止！
    InstanceId: tea.String("r-xxx"),         // 禁止！
}
```

**数值类型辅助函数**：对于 `int32/int` 字段，推荐使用 `atoiOr` 兜底：

```go
func atoiOr(s string, def int) int {
    if s == "" { return def }
    n, err := strconv.Atoi(s)
    if err != nil { return def }
    return n
}

// 用法
DataNodeAmount: tea.Int32(int32(atoiOr(os.Getenv("DATA_NODE_AMOUNT"), 3))),
```

**Endpoint 也从 env 拼接**（单 region 产品例外）：

```go
// ✅ 标准做法 — 多数产品
c, err := NewClient("elasticsearch." + os.Getenv("ALIBABA_CLOUD_REGION_ID") + ".aliyuncs.com")

// ⚠️ 单 region 产品例外（DAS、ActionTrail、审计类等）
// endpoint 固定为 cn-shanghai，但 RegionId 字段仍从 env 读取
c, err := NewClient("das.cn-shanghai.aliyuncs.com")
// (在文件头部加注释说明这是产品约束)
```

## 4. `go.mod` 模板

```go
module github.com/aliyun-skills/alicloud-[product]-ops/assets/code-snippets

go 1.21

require (
    github.com/alibabacloud-go/darabonba-openapi/v2 v2.x.x
    github.com/alibabacloud-go/tea v1.x.x
    github.com/alibabacloud-go/<product-package>/<version> v<x.x.x>   // 必须与 OpenAPI 版本一致
)
```

> `go.sum` 由 `go mod tidy` 生成；不在模板中提供。
> **SDK 包版本必须与 skill frontmatter 的 `api_profile` 字段一致**（如 `RDS 2014-08-15`
> 对应 `rds-20140815/v2/client`）。

## 5. `README.md` 模板

```markdown
# alicloud-[product]-ops Code Snippets

> **Generated by alicloud-skill-generator.** 这些片段仅在该 skill 的
> `cli_applicability: sdk-only` 时生成。CLI 能覆盖的 skill 请直接使用
> `aliyun` CLI，不要从这里复制代码。

## 前置条件

- Go 1.21+ (推荐 1.24+)
- 环境变量：
  - `ALIBABA_CLOUD_ACCESS_KEY_ID`
  - `ALIBABA_CLOUD_ACCESS_KEY_SECRET`
  - `ALIBABA_CLOUD_REGION_ID`（默认 `cn-hangzhou`）

## 使用方式

```bash
cd alicloud-[product]-ops/assets/code-snippets

# 第一次：拉取依赖
go mod tidy

# 运行某个 operation
ALIBABA_CLOUD_REGION_ID=cn-hangzhou \
  go run create_<resource>.go \
    -DBInstanceClass rds.mysql.s1.large \
    -DBInstanceStorage 20
```

> ⚠️ 上述 flag 是示例；实际参数请参考 [references/api-sdk-usage.md](../../references/api-sdk-usage.md)。

## Snippet 列表

| File | Operation | Required Args |
|------|-----------|---------------|
| `create_<resource>.go` | `<OperationId>` | `<arg1>`, `<arg2>` |
| `describe_<resource>.go` | `<OperationId>` | `<arg1>` |
| `modify_<resource>.go` | `<OperationId>` | `<arg1>`, `<arg2>` |
| `delete_<resource>.go` | `<OperationId>` | `<arg1>` |

## 注意事项

1. **绝不要在脚本中硬编码 AK/SK** — 始终用环境变量
2. **错误处理**：所有 snippet 遇到错误立即 `log.Fatalf` 退出，并保留 SDK 原始错误信息（含 RequestId）
3. **可执行性**：每个 `.go` 都应能 `go run` 成功；不依赖其它 snippet 的内部函数（除 `main.go` 的 `NewClient`）
4. **升级 SDK**：升级 package 版本时同步修改 `go.mod` 与 `references/integration.md`
```

## 6. Agent 生成流程

当 `cli_applicability: sdk-only` 时，generator 按以下顺序生成：

1. **解析 OpenAPI** 提取 `operations[]`，确定每个 Operation 的
   `OperationId / RequestType / MethodName / 必填参数 / Response 关键字段`
2. **创建 `assets/code-snippets/` 目录**，写入 `main.go` (Client 工厂) + `go.mod`
3. **遍历 `operations[]`**，每个 operation 生成一个 `<verb>_<resource>.go`：
   - 头部注释填 `<OperationId>` / `<MethodName>` / OpenAPI doc URL
   - Request 构造填必填参数 + 1-2 个常用可选参数
   - 末尾注释 `// $.<JsonPath> -> <用途>` 列出关键输出路径
4. **生成 `README.md`**，自动填入 Snippet 列表（基于上一步的元数据）
5. **运行验证**（如沙箱可用）：
   ```bash
   cd assets/code-snippets
   go mod tidy && go vet ./...
   ```
   若失败 → 修复 `go.mod` 版本号或 import 路径，重试

## 7. 与 SKILL.md 的关系

| Skill 中位置 | 用途 |
|---|---|
| `SKILL.md` Reference Directory | 列 `assets/code-snippets/README.md` 链接 |
| `SKILL.md` "Common JSON Paths" | 列出 snippets 输出关键字段的 JSON 路径 |
| `references/api-sdk-usage.md` | 详细说明 SDK Request/Response 字段（snippet 注释里只放指针） |
| `references/integration.md` | 标注 "sdk-only operation 参考 assets/code-snippets/" |

> **避免重复**：snippet 内联代码不放详细参数说明；只放可执行骨架。详细字段说明留在 `references/api-sdk-usage.md`。

## 8. 反模式 (Anti-Patterns)

| Anti-Pattern | 修正 |
|---|---|
| ❌ 所有 operation 挤在一个 `main.go` | ✅ 每个 operation 独立 `.go` 文件 |
| ❌ 在 snippet 中复制大段 docstring | ✅ 只放 `//` 行级注释 + 关键 JSON path 指针 |
| ❌ 硬编码 AK/SK 或在日志打印 SK | ✅ 始终从 `os.Getenv()` 读，禁止任何 SK 回显 |
| ❌ 缺少 `go.mod` 导致 `go run` 失败 | ✅ 必须有 `go.mod` + `go.sum` (后者由 tidy 生成) |
| ❌ snippet 引用了不存在的 SDK 包 | ✅ go.mod 必须与 `references/integration.md` 同步 |
| ❌ `cli-first` / `dual-path` skill 也生成 snippets | ✅ 仅 `sdk-only` (或 `cli-only` 含 SDK 操作) 触发 |
| ❌ snippet 编译失败但未被验证 | ✅ generator 必须执行 `go vet ./...` 验证 |
