# NAT Fault Pattern Knowledge Base

> **Purpose:** Fault pattern library for NAT Gateway. Each pattern follows the standardized schema.

## NAT-001 — SNAT 连接耗尽

| 属性 | 内容 |
|------|------|
| 触发指标 | MaxConnection > 80% 上限 |
| 触发阈值 | 连接 > 80% 上限持续 10 min |
| 典型特征 | 内网实例无法访问外部服务，SNAT 超时 |
| 关联指标 | DropConnection 非零、OutRatePercent 正常 |
| 根因 | 1. 单 EIP 连接容量 ~30K 已达上限 2. 连接泄漏 3. 并发请求过多 |
| 诊断步骤 | 1. DescribeNatGateways 查状态 2. 统计连接分布 3. 查异常实例 |
| 修复方案 | 1. 添加更多 EIP 绑定到 NAT 2. 优化连接池和超时 |
| 预防措施 | 连接数 70% 预警、EIP 扩容预案 |

## NAT-002 — DNAT 端口映射不通

| 属性 | 内容 |
|------|------|
| 触发指标 | External IP:Port 无响应 |
| 触发阈值 | DNAT 目标端口不可达 > 3 min |
| 典型特征 | 外部无法通过 DNAT 访问内部服务，但内部服务正常 |
| 关联指标 | Internal IP 健康，外部 IP 无入流量 |
| 根因 | 1. DNAT 配置错误 2. 后端安全组阻止 3. ECS 防火墙阻止 |
| 诊断步骤 | 1. DescribeForwardTableEntries 查配置 2. 查 ECS 安全组 3. 从 VPC 内部访问验证 |
| 修复方案 | 修正 DNAT 配置、调整安全组 |
| 预防措施 | DNAT 配置测试流程、变更后验证 |

## NAT-003 — NAT 带宽饱和

| 属性 | 内容 |
|------|------|
| 触发指标 | OutRatePercent > 90% 持续 5 min |
| 触发阈值 | 带宽 > 90% 规格限制 |
| 典型特征 | 出站请求超时，内部服务调用外部超时 |
| 关联指标 | NewConnection 下降（超时导致）、DropPacketTX 上升 |
| 根因 | 1. 大文件上传 2. 批量数据同步 3. 爬虫对外请求 |
| 诊断步骤 | 1. FlowLog 分析外部目标 2. 统计 Top 内部源 3. 确认业务是否异常 |
| 修复方案 | 1. 升级 NAT 规格 2. 限流 |
| 预防措施 | 带宽 80% 预警、业务流量基线 |

## NAT-004 — EIP 解绑后 SNAT 失效

| 属性 | 内容 |
|------|------|
| 触发指标 | SNAT 流量突降至 0 |
| 触发阈值 | SNAT 流量突降 > 90% |
| 典型特征 | 内网实例突然无法访问互联网 |
| 关联指标 | EIP 解绑事件、SNAT 流量同时归零 |
| 根因 | 1. 误操作解绑 EIP 2. EIP 被其他服务占用 3. 自动脚本逻辑错误 |
| 诊断步骤 | 1. DescribeSnatTableEntries 查状态 2. DescribeEipAddresses 查 EIP 绑定状态 |
| 修复方案 | 1. 重新绑定 EIP 2. 验证 SNAT 恢复 |
| 预防措施 | EIP 解绑操作审批、SNAT 依赖告警 |

## NAT-005 — NAT 网关规格不匹配

| 属性 | 内容 |
|------|------|
| 触发指标 | 规格 CPU > 80% 持续 30 min |
| 触发阈值 | CPU > 80% 或 CU > 规格上限 |
| 典型特征 | 延迟增加、连接建立慢、部分超时 |
| 关联指标 | 延迟指标上升、吞吐量接近上限 |
| 根因 | 1. 业务增长后规格未升级 2. 规格选型过低 3. 突发流量 |
| 诊断步骤 | 1. 查历史 CPU/连接趋势 2. 对比规格限制 |
| 修复方案 | 升级 NAT 规格（Small → Medium → Large → XLarge） |
| 预防措施 | 定期规格评估、自动扩缩容 |

## Cross-Product — NAT 连接耗尽 → RDS 无法拉取外部依赖

**场景：** NAT 连接耗尽 → 内部服务无法连接外网拉取镜像/依赖 → 部署失败

| 级联阶段 | 时间 | 现象 | Skill 负责 |
|----------|------|------|------------|
| T0 | 00:00 | NAT 连接耗尽，SNAT 超时 | `alicloud-nat-ops` |
| T1 | +1 min | ACK 集群无法拉取镜像 | `alicloud-ack-ops` |
| T2 | +5 min | 部署失败，Pod Pending | `alicloud-ack-ops` |
| T3 | +10 min | HPA 无法扩容，服务不可用 | `alicloud-slb-ops` |

**诊断顺序：** ACK Pod Pending → ACK 报 ImagePull → NAT SNAT → NAT 连接数

## NAT-006 — 闲置 NAT 网关浪费成本

| 属性 | 内容 |
|------|------|
| 触发指标 | SNAT 条目数 = 0 AND DNAT 条目数 = 0 持续 7 天 |
| 触发阈值 | 无 SNAT/DNAT 规则的 NAT Gateway 存在超过 7 天 |
| 典型特征 | NAT Gateway 持续产生实例费和 EIP 费，但无实际流量 |
| 关联指标 | CU 利用率 = 0，ActiveConnection = 0 |
| 根因 | 1. 项目下线后未清理 NAT 2. 测试环境遗忘 3. 迁移后旧 NAT 未释放 |
| 诊断步骤 | 1. DescribeNatGateways 列出所有 NAT 2. 逐一检查 SNAT/DNAT 条目数 3. 检查 EIP 流量 |
| 修复方案 | 1. 确认 NAT 无业务依赖 2. 释放关联 EIP 3. 删除 NAT Gateway |
| 预防措施 | 定期 FinOps 巡检、NAT 资源标签管理、项目下线检查清单 |

## NAT-007 — NAT 规格与负载不匹配（过度配置）

| 属性 | 内容 |
|------|------|
| 触发指标 | CU 利用率 < 20% 持续 30 天 |
| 触发阈值 | CU 平均利用率 < 20% 或带宽利用率 < 30% |
| 典型特征 | NAT 规格为 Large/XLarge 但实际流量远低于容量 |
| 关联指标 | MaxConnection 远低于规格上限、OutRatePercent 长期低位 |
| 根因 | 1. 初始选型过大 2. 业务缩减后未降配 3. 保守配置策略 |
| 诊断步骤 | 1. 查 30 天 CU/带宽趋势 2. 对比规格限制 3. 评估降配空间 |
| 修复方案 | 1. 降配 NAT 规格（Large → Medium → Small） 2. 切换计费模式为 PayByActualUsage |
| 预防措施 | 月度规格审查、CU 利用率告警 < 20% |

## NAT-008 — DNAT 高危端口暴露

| 属性 | 内容 |
|------|------|
| 触发指标 | DNAT 条目中 ExternalPort 为 22/3306/6379/3389/27017 |
| 触发阈值 | 任何高危端口通过 DNAT 映射到公网 |
| 典型特征 | SSH/数据库/Redis 端口直接暴露在互联网上 |
| 关联指标 | 入流量异常、安全扫描告警、暴力破解尝试 |
| 根因 | 1. 临时调试后遗忘关闭 2. 缺乏安全审查流程 3. 配置错误 |
| 诊断步骤 | 1. DescribeForwardTableEntries 扫描所有 DNAT 2. 筛选高危端口 3. 检查安全组 |
| 修复方案 | 1. 立即删除高危端口 DNAT 条目 2. 使用 VPN/Bastion 替代 SSH 3. 数据库仅内网访问 |
| 预防措施 | DNAT 创建时安全审查、定期安全巡检、高危端口创建告警 |

## NAT-009 — SNAT 源范围过于宽泛

| 属性 | 内容 |
|------|------|
| 触发指标 | SNAT 条目 SourceCIDR = 0.0.0.0/0 |
| 触发阈值 | SNAT 源 CIDR 为 0.0.0.0/0 或超大范围 |
| 典型特征 | 所有 VPC 内实例均可通过 NAT 访问互联网，缺乏精细控制 |
| 关联指标 | 出站流量异常增大、安全审计不合规 |
| 根因 | 1. 初始配置为快速开通 2. 缺乏最小权限原则 3. 迁移遗留 |
| 诊断步骤 | 1. DescribeSnatTableEntries 检查所有 SNAT 2. 筛选 0.0.0.0/0 源 3. 评估实际需要的 CIDR |
| 修复方案 | 1. 删除宽泛 SNAT 条目 2. 创建精确 CIDR 的 SNAT（如 10.0.1.0/24） 3. 使用 vSwitch 级别 SNAT |
| 预防措施 | SNAT 创建时 CIDR 审查、安全基线检查、最小暴露原则 |

## NAT-010 — EIP 计费模式不匹配导致成本浪费

| 属性 | 内容 |
|------|------|
| 触发指标 | EIP 流量模式与计费模式不匹配 |
| 触发阈值 | PayByBandwidth 但带宽利用率 < 30%，或 PayByTraffic 但流量稳定高 |
| 典型特征 | EIP 费用异常偏高或与实际使用不匹配 |
| 关联指标 | 带宽利用率趋势、流量 GB 趋势 |
| 根因 | 1. 初始计费模式选择不当 2. 业务模式变更后未调整 3. 缺乏成本审查 |
| 诊断步骤 | 1. 查 EIP 计费模式 2. 分析 30 天流量/带宽趋势 3. 计算两种模式成本对比 |
| 修复方案 | 1. 低利用率 PayByBandwidth → PayByTraffic 2. 稳定高流量 PayByTraffic → PayByBandwidth 3. 多 EIP → 共享带宽包 |
| 预防措施 | 月度计费模式审查、EIP 成本告警、CBWP 优化评估 |

## NAT-011 — 敏感系统变更未遵守变更窗口

| 属性 | 内容 |
|------|------|
| 触发指标 | L0/L1 NAT Gateway 变更发生在非变更窗口时间 |
| 触发阈值 | 变更操作时间不在批准的变更窗口内 |
| 典型特征 | 核心生产 NAT 在业务高峰期被修改，导致服务中断 |
| 关联指标 | ActionTrail 事件时间、变更窗口配置 |
| 根因 | 1. 缺乏变更窗口强制检查 2. 紧急变更未走审批 3. 自动化脚本无窗口校验 |
| 诊断步骤 | 1. 查 ActionTrail 中 NAT 变更事件时间 2. 对比变更窗口策略 3. 检查是否有 CAB 审批记录 |
| 修复方案 | 1. 立即评估变更影响 2. 如有异常执行回滚 3. 补办 CAB 审批 4. 强化变更窗口自动化检查 |
| 预防措施 | 变更窗口自动化校验、CAB 审批流程集成、紧急变更分级授权 |

## NAT-012 — 敏感系统变更缺少回滚预案

| 属性 | 内容 |
|------|------|
| 触发指标 | L0/L1 NAT Gateway 变更后异常，无回滚方案 |
| 触发阈值 | 变更失败且无配置快照或回滚步骤 |
| 典型特征 | NAT 规格降配后连接数超限，无法快速恢复 |
| 关联指标 | 变更后 DropConnection 飙升、ActiveConnection 骤降 |
| 根因 | 1. 变更前未捕获配置快照 2. 未制定回滚步骤 3. 回滚时间未评估 |
| 诊断步骤 | 1. 检查变更前是否有快照 2. 查当前 NAT 配置与变更前差异 3. 评估手动恢复时间 |
| 修复方案 | 1. 从 ActionTrail 获取变更前配置 2. 手动重建 SNAT/DNAT 条目 3. 恢复 NAT 规格 4. 验证连通性 |
| 预防措施 | 变更前强制快照、回滚预案模板、自动化回滚触发 |

## NAT-013 — DR 备用 NAT 被误删导致容灾失效

| 属性 | 内容 |
|------|------|
| 触发指标 | 闲置 NAT Gateway 被删除，但实际为 DR 备用 |
| 触发阈值 | 闲置检测脚本标记 DR NAT 为可删除 |
| 典型特征 | 主 NAT 故障后，发现 DR NAT 已被成本优化删除 |
| 关联指标 | 闲置检测标记、DR 角色标签缺失 |
| 根因 | 1. NAT 未标记 Role=DR 2. 闲置检测未排除 DR 角色 3. 成本优化未考虑容灾架构 |
| 诊断步骤 | 1. 检查已删除 NAT 是否有 DR 标签 2. 评估当前容灾架构完整性 3. 确认主 NAT 的 HA 保障 |
| 修复方案 | 1. 立即重建 DR NAT Gateway 2. 重新配置 SNAT/DNAT 3. 标记 Role=DR 4. 验证 DR 切换流程 |
| 预防措施 | DR 资源强制标签、闲置检测排除 DR 角色、删除前 CAB 审批 |

## NAT-014 — L0 DNAT 违规暴露数据库端口

| 属性 | 内容 |
|------|------|
| 触发指标 | L0 NAT Gateway 的 DNAT 条目暴露数据库端口 |
| 触发阈值 | L0 NAT 上存在 3306/6379/27017 等数据库端口 DNAT |
| 典型特征 | 核心生产数据库通过 DNAT 直接暴露在公网 |
| 关联指标 | 安全扫描告警、异常入流量、暴力破解尝试 |
| 根因 | 1. 紧急调试临时开放 2. 安全审查未区分敏感级别 3. 缺乏 L0 专属安全基线 |
| 诊断步骤 | 1. 扫描 L0 NAT 的所有 DNAT 条目 2. 筛选数据库端口 3. 检查安全组是否有限制 |
| 修复方案 | 1. **立即**删除数据库端口 DNAT（L0 紧急响应 < 15 min） 2. 配置 VPC 对等连接或 PrivateLink 3. 安全事件通报 4. 事后渗透测试 |
| 预防措施 | L0 DNAT 双人审批、数据库端口创建自动拦截、每日安全巡检 |
