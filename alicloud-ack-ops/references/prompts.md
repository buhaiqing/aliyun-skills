# Prompt Examples — Alibaba Cloud ACK

This document provides **natural language prompt examples** that users can use to interact with the `alicloud-ack-ops` skill. Each example demonstrates a common operational scenario. Users can copy and adapt these prompts directly.

> **使用方式:** 将下面任意一条提示词发给 Agent，Skill 会自动识别场景、收集缺失参数、执行操作并返回结果。
> **变量约定:** `{变量}` 表示需要用户提供的信息，Agent 会在执行前主动询问。


## 使用技巧 (Usage Tips)

### 简短描述即可

不需要精确说出参数名，Agent 会根据 Skill 文档自动映射：

| 你的描述 | 会被识别为 |
|---------|-----------|
| "杭州" | `RegionId: cn-hangzhou` |
| "托管版集群" | `ClusterType: ManagedKubernetes` |
| "节点池扩容" | `ScaleOutNodePool` |
| "Pod 一直 Pending" | 触发 Pod 诊断流程 |
| "CrashLoopBackOff" | 进入 Pod 健康检查诊断 |

### 可以一次说多件事

```
帮我巡检集群 c-xxx，检查节点状态、Pod 异常、资源利用率，生成报告
```

### 不用记参数名

Agent 会自动询问缺失的信息。你只需要描述"要做什么"和"大概什么配置"。

### 安全操作有确认

涉及删除集群、驱逐节点、删除 Pod 等操作时，Agent 会主动要求二次确认，防止误操作。

### 多指标巡检自动评分

执行多指标巡检时，Agent 会：
1. 自动收集 CPU、内存、磁盘、Pod 状态等指标
2. 计算健康度评分（0-100）
3. 生成优化建议清单
4. 提供可执行的修复命令

### 告警诊断自动关联

收到告警时，Agent 会：
1. 自动关联相关指标（如 CPU 告警 → 检查节点、Pod、内存）
2. 匹配故障模式库
3. 提供根因分析和修复建议
4. 如需跨 Skill 协作，自动委托（如 ECS 磁盘问题 → 委托 ECS Skill）
