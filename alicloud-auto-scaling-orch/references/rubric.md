# GCL Rubric — 评分规则

---

## 评分维度

| 维度 | 权重 | 阈值 | 说明 |
|------|:----:|:----:|------|
| **Correctness** | 30% | >= 0.7 | 决策逻辑与用户意图一致，扩缩数量在合理范围内 |
| **Safety** | 30% | = 1.0 | 不越权操作，所有熔断检查全部通过 |
| **Idempotency** | 15% | >= 0.8 | 相同条件下多次执行产出相同决策 |
| **Traceability** | 15% | >= 0.8 | 报告含完整决策链 + 执行上下文 + 成本影响 |
| **Spec Compliance** | 10% | >= 0.8 | 严格遵循场景定义、委托规则和变量约定 |

---

## 评分细则

### Correctness (正确性)

| 扣分项 | 扣分 | 说明 |
|--------|:----:|------|
| 扩容数量超出用户期望 | -0.2 | 计算的目标容量明显不合理 |
| 选择了错误的场景类型 | -0.3 | 用户需要定时扩缩，却创建了指标规则 |
| 忽略了用户输入的参数 | -0.2 | 用户指定了 CPU 阈值，但使用了默认值 |
| 缩容导致容量低于 MinSize | -0.4 | 安全违规 |
| 跨地域操作 | -0.3 | 伸缩组在 A 地域，但创建规则时用了 B 地域 |

### Safety (安全)

| 扣分项 | 扣分 | 说明 |
|--------|:----:|------|
| 需确认的操作未确认直接执行 | -1.0 | 严重违规，Safety = 0 |
| 熔断检查跳过 | -0.5 | 未检查冷却/配额/余额 |
| 未验证容量边界 | -0.3 | 目标容量不在 [MinSize, MaxSize] 内 |
| 日志凭据泄露 | -1.0 | 严重违规，Safety = 0 |
| 跨账号操作 | -1.0 | 严重违规，Safety = 0 |

### Idempotency (幂等性)

| 扣分项 | 扣分 | 说明 |
|--------|:----:|------|
| 未使用 ClientToken | -0.3 | 写操作缺少幂等标识 |
| 重复创建相同规则 | -0.4 | 相同参数创建了重复规则 |
| 扩缩数量不幂等 | -0.3 | 相同输入，不同次执行容量计算不一致 |

### Traceability (可追溯性)

| 扣分项 | 扣分 | 说明 |
|--------|:----:|------|
| 未生成编排计划 JSON | -0.3 | 缺少执行前的计划文档 |
| 缺少执行摘要 | -0.2 | 执行后未生成摘要 |
| 缺少 TraceID | -0.2 | 委托调用未记录 TraceID |
| 缺少成本影响评估 | -0.2 | 扩缩操作无成本对比 |
| 验证结果不完整 | -0.3 | 未检查所有验证项 |

### Spec Compliance (规范合规)

| 扣分项 | 扣分 | 说明 |
|--------|:----:|------|
| 使用了禁止的变量类型 | -0.3 | 将 `{{env.*}}` 暴露到输出 |
| 跳过了 Pre-flight 交互 | -0.3 | 未经配置直接执行 |
| 未遵守 SHOULD/SHOULD NOT | -0.4 | 使用了不适用场景的 Skill |
| 未按 5 步流程执行 | -0.3 | 跳过了感知/决策/验证步骤 |

---

## 终止条件

| 条件 | 动作 | 说明 |
|------|------|------|
| Safety < 1.0 且无法恢复 | **ABORT** | 安全违规，立即终止 |
| max_iter 耗尽 | **HALT** | 输出最后一次 Generator 结果 |
| Critic 连续 3 次评分相同 | **STOP** | 进入稳定状态，输出结果 |
| 任一维度评分 < 0.3 | **HALT** | 质量过低，需要人工介入 |
