## 2. 容量计算公式

### 2.1 指标驱动 (S1, S4)

```
期望容量 = ceil(当前实例数 × 当前负载率 / 目标利用率)

其中:
  当前负载率 = max(CPU利用率, 内存利用率)  (S4 取加权)
  目标利用率 = TargetTracking 的 TargetValue

示例:
  当前 3 台, CPU=75%, 目标=60%
  → ceil(3 × 0.75 / 0.60) = ceil(3.75) = 4 台 → 扩容 1 台
```

### 2.2 定时任务 (S2)

```
直接使用用户指定的 DesiredCapacity:
  扩容时间 → DesiredCapacity = 5
  缩容时间 → DesiredCapacity = 1

不做计算，仅做验证 (目标值在 MinSize..MaxSize 范围内)
```

### 2.3 预测性 (S3)

```
由 ESS 内置算法自动计算，不手动干预

预估容量 = PredictiveScalingRule 自动输出的 MaxOverridePredictiveValue
Agent 只需确认: 目标利用率 + Buffer 值是否合理
```

### 2.4 大促保障 (S5)

```
期望容量 = ceil(当前实例数 × 历史峰值CPU × 安全系数 / 目标利用率)

参数参考:
  历史峰值: 取过去 30 天 P95 CPU
  安全系数: 1.5 (标准) / 2.0 (高保障) / 3.0 (关键业务)
  示例: 5 × 0.85 × 1.5 / 0.60 = ceil(10.625) = 11 → 预扩至 12 台

渐缩公式:
  第 i 步期望 = ceil(活动期容量 - (活动期容量 - 基线条目) × i / 总步数)
  示例: 12 → 9 → 6 → 4 → 2 (4 步渐缩)
```

### 2.5 闲置回收 (S6)

```
期望容量 = MinSize (最小保障)

无需公式计算，直接缩至 MinSize。
注意: 缩容前必须有 24h 预警通知 + 用户确认 (或白名单)
```

## 3. 指标采集规范

### 3.1 CMS Namespace 映射

| 资源类型 | Namespace | 关键 Metric |
|---------|-----------|-------------|
| ECS | `acs_ecs_dashboard` | `CpuUtilization`, `memory_usedutilization`, `InternetInRate`, `InternetOutRate` |
| SLB | `acs_slb_dashboard` | `InstanceNewConnection`, `InstanceActiveConnection`, `InstanceQPS` |
| RDS | `acs_rds_dashboard` | `CpuUsage`, `ConnectionUsage`, `MemoryUsage` |
| Redis | `acs_kvstore` | `CpuUsage`, `MemoryUsage`, `ConnectionUsage` |

### 3.2 采集窗口

| 场景 | 采集窗口 | 粒度 | 用途 |
|------|---------|:----:|------|
| S1 实时决策 | 最近 1h | 5min | 判断当前是否需要扩缩 |
| S3 预测 | 14 天 | 1h | 分析周期性 |
| S5 大促规划 | 30 天 | 1h | 计算历史峰值 |
| S6 闲置判定 | 7 天 (可配置) | 1h | 判定低负载持续时长 |

### 3.3 异常值处理

| 情况 | 处理方式 |
|------|---------|
| 指标缺失 (No Data) | 跳过该周期，取上一个有效值 |
| 指标为 0 | 保留 (可能是真实值) |
| 指标 > 100 | 丢弃 (数据异常) |
| 实例刚启动 < 10min | 跳过 (数据不稳定) |

## 4. 冷却与防抖动

### 4.1 冷却时间公式

```
冷却时间 = max(实例预热时间 × 1.5, 固定冷却_300s)

实例预热时间 = EstimatedInstanceWarmup (new)
             或 max(启动时间 × 2, 300s) (无配置时)
```

### 4.2 防抖动规则

| 规则 | 说明 | 动作 |
|------|------|------|
| 同向最短间隔 | 两次扩容间隔 > 冷却时间 | 否则忽略 |
| 反向最短间隔 | 扩→缩间隔 > 冷却时间 × 2 | 否则忽略 |
| 方向反转检测 | 1h 内扩缩反转 > 3 次 | 熔断 1h + 分析报告 |
| 小波动过滤 | 变化 < 10% 且持续 < 5min | 忽略 |
| 连续触发上限 | 连续同向触发 > 3 次 | 每次 step 递减 50% |

## 5. 策略决策优先级

当多个策略同时生效时，按此优先级决策：

```
1. 大促弹性保障 (S5)             → 最高优先级，覆盖其他所有策略
2. 闲置资源回收 (S6)              → 仅在 S5 不活跃时生效
3. 复合多指标 (S4)                → 次高，覆盖 S1/S2
4. CPU 指标驱动 (S1)              → 标准
5. 预测性扩缩 (S3)                → 与 S1 互补
6. 定时业务周期 (S2)              → 基础保底
```

**冲突处理**：当多个策略的目标容量冲突时，取最大值。

---

## 6. 安全边界校验

每项决策在执行前必须通过以下校验：

| 校验项 | 检查条件 | 失败处理 |
|--------|---------|---------|
| 容量边界 | 目标容量 ∈ [MinSize, MaxSize] | 截断到边界值 + 警告 |
| 配额限制 | 目标容量 ≤ 地域配额 | 提示当前配额使用率 |
| 实例类型 | 目标实例类型在伸缩配置中可用 | 提示可用规格列表 |
| 余额充足 | 账户余额 > 预估费用 × 2 | HALT + 余额不足告警 |
| 无审批拒绝 | 需确认的操作已获得确认 | HALT + 等待确认 |
