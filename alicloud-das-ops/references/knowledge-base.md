# DAS Fault Pattern Knowledge Base

> **Purpose:** Fault pattern library for DAS (数据库自治服务). Each pattern follows the standardized schema.

## DAS-001 — 自治诊断误报

| 属性 | 内容 |
|------|------|
| 触发指标 | `CreateDiagnosticReport` 返回的建议评分异常 |
| 触发阈值 | 诊断报告误判根因 |
| 典型特征 | DAS 建议的修复方案无效或风险高 |
| 关联指标 | 诊断得分高但实际问题未解决 |
| 根因 | 1. 基线数据不准确 2. 异常模式匹配错误 3. 统计信息过期 |
| 诊断步骤 | 1. 手动分析 SQL 执行计划 2. 对比 DAS 建议 3. 验证基线数据 |
| 修复方案 | 1. 手动修正 2. 调整 DAS 诊断参数 |
| 预防措施 | DAS 建议需验证后执行、定期校准基线 |

## DAS-002 — 自治事件未触发

| 属性 | 内容 |
|------|------|
| 触发指标 | 性能明显下降但 DAS 无诊断事件 |
| 触发阈值 | 性能下降 > 30% 但诊断未触发 |
| 典型特征 | 数据库慢查询/延迟增加，DAS 无告警 |
| 关联指标 | 慢查询日志增多但 DAS 未报告 |
| 根因 | 1. 诊断阈值配置过高 2. 监控粒度不足 3. 未启用必要功能 |
| 诊断步骤 | 1. 检查 DAS 配置 2. 查诊断阈值 3. 确认所有诊断功能是否启用 |
| 修复方案 | 1. 调低阈值 2. 启用 SQL 洞察 3. 开启全量诊断 |
| 预防措施 | 定期审核 DAS 配置、诊断功能全覆盖 |

## DAS-003 — DAS 服务不可用

| 属性 | 内容 |
|------|------|
| 触发指标 | DAS API 返回 `ServiceUnavailable` |
| 触发阈值 | DAS 服务不可用 > 5 min |
| 典型特征 | 创建诊断报告/查看分析失败 |
| 关联指标 | CMS 降级，无法诊断数据库问题 |
| 根因 | 1. DAS 服务维护 2. 权限问题 3. 网络不通 |
| 诊断步骤 | 1. 查 DAS 服务状态 2. 确认 RAM 权限 3. 网络连通性测试 |
| 修复方案 | 1. 降级到手动诊断 2. 等待 DAS 恢复 |
| 预防措施 | 备用诊断方案（手动 SQL 分析） |

## Cross-Product — DAS 不可用 → RDS 问题无法自动诊断

**场景：** DAS 故障 → RDS 问题只能人工排查 → 诊断时间大幅延长

| 级联阶段 | 时间 | 现象 | Skill 负责 |
|----------|------|------|------------|
| T0 | 00:00 | DAS 不可用 | `alicloud-das-ops` |
| T1 | +5 min | RDS 问题出现，无法自动诊断 | `alicloud-rds-ops` |
| T2 | +15 min | 人工排查替代，MTTR 增加 | DBA 团队 |

**诊断顺序：** RDS 告警 → DAS 不可用 → 降级到手动 `SHOW PROCESSLIST` + `EXPLAIN` + 慢 SQL 日志分析
