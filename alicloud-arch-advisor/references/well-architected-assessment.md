# Well-Architected Assessment — alicloud-arch-advisor 自我评估

> 本文件对 alicloud-arch-advisor Skill 本身进行 Well-Architected Framework 五支柱评估（元认知层）。评估对象是该 Skill 的设计、实现和使用方式是否符合 WAF 最佳实践。

---

## Security (安全)

| 原则 | 实施方式 |
|------|---------|
| **最小权限** | 本 Skill 为纯只读 Skill，不执行任何 OpenAPI 写操作。所有数据通过委托下游只读 Skill 获取 |
| **凭证隔离** | 严格遵守 `{{env.*}}` ONLY 规范，NEVER 在报告/日志中暴露 AK/SK |
| **上下文隔离** | 委托下游 Skill 时，仅传递所需的最小上下文，不过度暴露用户信息 |
| **审计追踪** | 每份报告记录数据源时间戳，支持回溯：哪些数据在何时从哪个 Skill 获取 |
| **安全边界** | Safety Gate 硬约束：任何写入性建议在报告中标注为"操作建议"，由用户通过对应产品 Skill 执行 |

### 评估

| 维度 | 评分 | 备注 |
|:----|:---:|------|
| 凭证管理 | ★★★★★ | 纯只读，无任何凭证使用风险 |
| 数据保护 | ★★★★★ | 不存储用户数据，报告生成即完成 |
| 审计能力 | ★★★★☆ | 报告含数据源记录，未来可增强到事件级审计 |
| 最小权限 | ★★★★★ | 不持有任何 OpenAPI 权限 |

## Reliability (可靠性)

| 原则 | 实施方式 |
|------|---------|
| **三模式标准化流程** | 每种模式有明确的 Phase 1/2/3 流程，流程可重复 |
| **数据源降级策略** | 当数据源 Skill 不可用时，采用用户描述 + confidence 标注的降级策略 |
| **输出验证** | 报告生成后由 Agent 或 GCL 验证逻辑一致性 |
| **无状态设计** | 每次执行独立，不依赖持久化状态或缓存 |
| **幂等性** | 相同输入 + 相同数据源状态 → 相同报告输出（Mode B 评分可复现） |

### 评估

| 维度 | 评分 | 备注 |
|:----|:---:|------|
| 流程标准化 | ★★★★★ | 三种模式各有标准化流程 |
| 降级容错 | ★★★★☆ | 数据源不可用时降级到用户描述，但评分质量下降 |
| 幂等性 | ★★★★★ | 纯只读分析，天然幂等 |
| 输出验证 | ★★★★☆ | 依赖 Agent 本身的验证能力 + GCL optional |

## Cost (成本)

| 原则 | 实施方式 |
|------|---------|
| **零资源消耗** | 本 Skill 自身不创建/持有任何云资源，运行成本为零 |
| **间接成本节省** | 通过 Mode B Cost 支柱评估，帮助用户识别闲置资源和规格优化机会 |
| **方案成本对比** | Mode C 中提供多方案月度成本估算和 TCO 对比 |
| **成本意识报告** | 所有建议标注预估实施成本和预期收益 |

### 评估

| 维度 | 评分 | 备注 |
|:----|:---:|------|
| 自身成本 | ★★★★★ | 无需额外云资源 |
| 帮用户省钱 | ★★★★☆ | 成本评估和建议可帮助用户优化支出 |
| 建议可执行性 | ★★★☆☆ | 建议需要用户通过产品 Skill 执行，非自动节省 |

## Efficiency (效率)

| 原则 | 实施方式 |
|------|---------|
| **三模式覆盖广泛** | 逆向分析 + 成熟度评估 + 方案推荐，覆盖主流架构咨询场景 |
| **报告模板复用** | 统一的 Markdown + JSON 报告格式，可被下游消费 |
| **并行数据采集** | 委托下游 Skill 时并行调用，减少用户等待时间 |
| **标准化流程** | 三模式标准化流程确保一致的服务质量 |
| **渐进式交付** | 先出摘要，用户深入时再展开详细分析 |

### 评估

| 维度 | 评分 | 备注 |
|:----|:---:|------|
| 场景覆盖 | ★★★★★ | 三种模式覆盖 > 90% 的架构咨询需求 |
| 交付效率 | ★★★★☆ | 数据采集并行化，报告生成 < 30s |
| 用户交互 | ★★★★☆ | 根据用户输入自动识别模式，无需用户指定 |
| 流程复用 | ★★★★★ | 跨用户、跨场景的一致报告格式 |

## Performance (性能)

| 原则 | 目标 | 说明 |
|:----|:----:|------|
| 模式识别 | < 3s | 基于用户输入关键词 + 上下文自动识别模式 |
| 数据采集 | < 15s | 委托下游 Skill 并行采集（受网络和 API 响应影响） |
| 五支柱评分 | < 5s | 基于已有数据+规则引擎计算 |
| 方案推荐 | < 10s | 模式匹配 + 模板选择（复杂场景可能需要更长思考时间） |
| 完整报告 | < 30s | 从接受到报告完成（不含用户交互时间） |

| 可扩展性 | ★★★★☆ | 支持增加新的 WAF 检查维度和场景模板 |
